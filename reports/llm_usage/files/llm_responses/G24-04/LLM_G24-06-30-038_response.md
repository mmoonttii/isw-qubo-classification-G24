Both bugs trace back to the same root cause: Streamlit's script-rerun model. Here are the fixes.

**Bug 1 (sidebar lag):** `_run_preprocessing`/`_run_feature_selection` mutate `session_state` but the function returns into the *same* script run — the sidebar was already drawn before the phase tab executed, so it's showing stale state until the next interaction forces a redraw.

**Bug 2 (state wipe):** `st.file_uploader` keeps returning the same `UploadedFile` object on *every* rerun for as long as it's attached to the widget — not just the run where you picked it. So once we add `st.rerun()` for bug 1, the very next script run hits `if uploaded is not None: _handle_upload(uploaded)` again, and `_handle_upload` unconditionally zeroes out every downstream phase flag.
